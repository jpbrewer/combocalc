# Alpha-Transfer Prompt

## Context

When a customer has finished reviewing a solution in the modal and clicks the Configure button, the selected solution needs to be reconciled, cleaned up, and saved to the Xano backend. This prompt describes that end-to-end flow.

The Configure button is identified in the DOM by `data-modal-configure="btn"`.

## What Needs to Change

### 1. Wire Up the Configure Button

Add a click handler to the element identified by `data-modal-configure="btn"`. This handler is responsible for the full sequence described below.

The current modal index (the index of the solution currently being displayed) is already tracked in the codebase. Use that value to identify which element in `window.comboSolutions` is being acted on.

### 2. Stage a Deep Copy

Before any reconciliation, create a deep copy of `comboSolutions[currentIndex]` and perform all subsequent operations on that staged copy. The live `comboSolutions` array must not be mutated at any point during this process.

### 3. Reconcile Rows and Cols on the Staged Copy

Resolve the final pane configuration on every build object in the staged copy's `build_objects`:

- If the staged copy's `muntins === true`, copy `suggested_rows` into `rows` and `suggested_cols` into `cols` on each build object where `suggested_rows` and `suggested_cols` are present. For build objects without those fields, leave `rows` and `cols` unchanged.
- If the staged copy's `muntins === false`, leave `rows` and `cols` as-is on all build objects.

### 4. Clean Up the Staged Copy

Remove the following fields from the staged copy before sending to Xano:

**From the solution object:**
- `muntins` — has served its purpose and should not reach the backend
- `building_block_svgs`
- `building_block_svgs_no_muntins`
- `assembly_svg`
- `assembly_svg_no_muntins`

These SVG fields will be regenerated by the SVG pipeline when the Beta codebase loads. There is no need to persist them.

**From every build object in `build_objects`:**
- `suggested_rows`
- `suggested_cols`

Delete these where present. Build objects that do not have them are unaffected.

### 5. Post to Xano

Send the staged copy as the POST body to:

```
https://api.transomsdirect.com/api:xyi0dc0X/bc_combo_solution_config
```

Use the existing pattern in the codebase for making Xano API calls. The payload is the staged copy as JSON. No fields should be stripped or transformed beyond the reconciliation and cleanup described above.

### 6. Error Handling

Follow the existing error handling pattern in the codebase. If the POST fails, surface the error appropriately and do not proceed silently.

## What Must Not Change

- The modal's open, close, and muntin toggle behavior
- The SVG pipeline and its caching behavior
- Any other button handlers or modal interactions
- The structure of `comboSolutions` for all other indices not currently being configured
